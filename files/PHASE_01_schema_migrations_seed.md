## Schema (Prisma) — 7 tables

> The brief lists 6 minimum tables. We add a 7th, `idempotency_records`, because the engine's idempotency
> contract (§6.4) requires a durable dedupe ledger. This is allowed ("minimum tables").

```prisma
enum InstanceStatus { pending running waiting failed completed }
enum NodeRunStatus  { pending running waiting failed completed }   // success maps to completed on persist
enum NodeType       { start api_call approval condition script end }
enum LogLevel       { debug info warn error }

model Workflow {
  id        String         @id @default(cuid())
  name      String
  version   Int            @default(1)
  status    String         @default("active")   // active | archived
  createdAt DateTime       @default(now())
  nodes     WorkflowNode[]
  edges     WorkflowEdge[]
  instances WorkflowInstance[]
}

model WorkflowNode {
  id         String   @id @default(cuid())
  workflowId String
  type       NodeType
  config     Json                    // jsonb
  position   Json                    // { x, y }
  workflow   Workflow @relation(fields: [workflowId], references: [id], onDelete: Cascade)
  outEdges   WorkflowEdge[] @relation("source")
  inEdges    WorkflowEdge[] @relation("target")
}

model WorkflowEdge {
  id           String   @id @default(cuid())
  workflowId   String
  sourceNodeId String
  targetNodeId String
  condition    Json?                  // jsonb, nullable (JSON-logic for Condition node edges)
  ordering     Int      @default(0)   // STABLE deterministic edge order (§6.1)
  workflow     Workflow @relation(fields: [workflowId], references: [id], onDelete: Cascade)
  source       WorkflowNode @relation("source", fields: [sourceNodeId], references: [id], onDelete: Cascade)
  target       WorkflowNode @relation("target", fields: [targetNodeId], references: [id], onDelete: Cascade)
  @@index([workflowId, sourceNodeId, ordering])
}

model WorkflowInstance {
  id            String         @id @default(cuid())
  workflowId    String
  status        InstanceStatus @default(pending)
  currentNodeId String?
  context       Json                       // jsonb accumulated payload
  startedBy     String                     // DEV_USER_ID
  createdAt     DateTime       @default(now())
  updatedAt     DateTime       @updatedAt
  workflow      Workflow       @relation(fields: [workflowId], references: [id])
  nodeRuns      NodeExecution[]
  logs          ExecutionLog[]
  @@index([status])
}

model NodeExecution {
  id         String        @id @default(cuid())
  instanceId String
  nodeId     String
  status     NodeRunStatus @default(pending)
  attempt    Int           @default(1)
  input      Json?
  output     Json?
  error      String?
  startedAt  DateTime?
  finishedAt DateTime?
  instance   WorkflowInstance @relation(fields: [instanceId], references: [id], onDelete: Cascade)
  logs       ExecutionLog[]
  @@index([instanceId])
  @@unique([instanceId, nodeId, attempt])   // makes a transition write idempotent
}

model ExecutionLog {
  id              String   @id @default(cuid())
  instanceId      String
  nodeExecutionId String?
  level           LogLevel @default(info)
  message         String
  data            Json?
  createdAt       DateTime @default(now())
  instance        WorkflowInstance @relation(fields: [instanceId], references: [id], onDelete: Cascade)
  nodeExecution   NodeExecution?   @relation(fields: [nodeExecutionId], references: [id], onDelete: Cascade)
  @@index([instanceId])
}

model IdempotencyRecord {
  key        String   @id            // sha256(instanceId|nodeId|attempt)
  instanceId String
  nodeId     String
  response   Json                    // cached external-write result
  createdAt  DateTime @default(now())
}
```

### External ERP target (`prisma/sql/001_external_schema.sql`)
```sql
CREATE SCHEMA IF NOT EXISTS erp_external;
CREATE TABLE IF NOT EXISTS erp_external.purchase_orders (
  id              text PRIMARY KEY,
  request_ref     text NOT NULL,
  amount          numeric(14,2) NOT NULL,
  supplier        text NOT NULL,
  idempotency_key text UNIQUE NOT NULL,    -- DB-level dedupe proof for the idempotency test
  created_at      timestamptz NOT NULL DEFAULT now()
);
```
Run this SQL inside the same migration step (Prisma `migrate dev` then `psql -f` the raw file via a
post-migrate script wired into `prisma`'s `seed`/scripts).

## Seed: Purchase Request Approval Flow (idempotent)
`seed.ts` upserts ONE workflow named `Purchase Request Approval` with these nodes/edges (stable `ordering`):

1. `start`        — Start, initial payload `{ amount, supplier, requestRef }`
2. `check_budget` — API Call → `GET {MOCK_ERP}/budget?amount={{ctx.amount}}` (returns `{ approved: bool }`)
3. `budget_gate`  — Condition: edge A (`ordering:0`, cond `{"==":[{"var":"check_budget.approved"},true]}`) → manager_approval; edge B (`ordering:1`, cond `true` fallback) → end_rejected
4. `manager_approval` — Approval (human)
5. `finance_approval` — Approval (human)
6. `create_po`    — API Call (external **write**) → `POST {MOCK_ERP}/po` with idempotency key; body templated from context
7. `notify_supplier` — Webhook (outbound POST to `{WEBHOOK_SINK}/notify`)
8. `end_ok`       — End (completed)
9. `end_rejected` — End (failed/closed, rejected branch)

Edges connect 1→2→3, 3⇒{4,9}, 4→5→6→7→8. All `condition` JSON stored as `jsonb`; non-condition edges have `condition: null`.

> `MOCK_ERP` and `WEBHOOK_SINK` resolve to the local mock server built in Phase 04 (`http://localhost:4000/mock-erp` and `/mock-sink`). Keep them in `.env`.
